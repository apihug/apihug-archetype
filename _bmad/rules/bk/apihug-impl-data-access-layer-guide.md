## Core Architecture

```
Proto Entity â†’ wire â†’ [Entity, DSL, Repository] â†’ Trait Extension â†’ Service
```

**Generated artifacts** (`src/generated/domain`):
- `XxxEntity` â€” Domain entity
- `XxxEntityDSL` â€” extends `DSL<XxxEntity>`
- `XxxEntityRepository` â€” extends `HopeJdbc<XxxEntity>`, `XxxEntityDSL`, `ListCrudRepository<XxxEntity, Long>`

**ListCrudRepository provides (NO need to redefine)**:

| Method                                            | Description       |
|---------------------------------------------------|-------------------|
| `save(entity)` / `saveAll(entities)`              | Insert or update  |
| `findById(id)` / `findAll()` / `findAllById(ids)` | Read operations   |
| `count()`                                         | Count all         |
| `deleteById(id)` / `delete(entity)`               | Delete operations |

Reference: https://docs.spring.io/spring-data/commons/reference/repositories/core-concepts.html

## 1. Trait Extension Patterns

> **IMPORTANT**: Trait extension inherits ALL methods from `ListCrudRepository` (save, findById, findAll, delete, etc.) and `HopeJdbc` (findOne, exists, count with Criteria). DO NOT redefine these.
>
> **ONLY add methods that don't exist** â€” custom queries, dynamic criteria, projections.

**Repository Traits:**
- Normal: `com.example.domain.feature.repository.UserRoleRepository`
- Trait:   `t.com.example.domain.feature.repository.UserRoleRepository` (prefix `t.`) in {trait_path}
- Trait interface is AUTO-GENERATED by wire â€” **FORBIDDEN to create manually**
- **FORBIDDEN** to remove any comment in the generated trait repository â€” MUST follow them

### Pattern 1: Spring Data Query Methods (PREFERRED for simple queries)

**Reference**: https://docs.spring.io/spring-data/relational/reference/jdbc/query-methods.html

Spring Data auto-implements methods based on naming convention:

```java
interface _UserRepository extends UserRepository {

    // === SINGLE FIELD QUERIES ===
    Optional<UserEntity> findByEmail(String email);
    Optional<UserEntity> findByUsernameAndTenantId(String username, Long tenantId);
    List<UserEntity> findByStatus(String status);
    List<UserEntity> findByTenantIdOrderByCreatedAtDesc(Long tenantId);

    // === COMPARISON OPERATORS ===
    List<UserEntity> findByCreatedAtAfter(Instant date);
    List<UserEntity> findByCreatedAtBetween(Instant start, Instant end);
    List<UserEntity> findByAgeGreaterThanEqual(Integer age);
    List<UserEntity> findByScoreLessThan(Double score);

    // === STRING MATCHING ===
    List<UserEntity> findByNameContaining(String name);      // LIKE '%name%'
    List<UserEntity> findByNameStartingWith(String prefix);  // LIKE 'prefix%'
    List<UserEntity> findByNameEndingWith(String suffix);    // LIKE '%suffix'
    List<UserEntity> findByNameLike(String pattern);         // LIKE pattern (wildcards in param)

    // === IN / NOT IN ===
    List<UserEntity> findByStatusIn(List<String> statuses);
    List<UserEntity> findByRoleNotIn(List<String> roles);

    // === NULL CHECKS ===
    List<UserEntity> findByDeletedAtIsNull();
    List<UserEntity> findByDeletedAtIsNotNull();

    // === BOOLEAN ===
    List<UserEntity> findByActiveTrue();
    List<UserEntity> findByActiveFalse();

    // === ORDERING ===
    List<UserEntity> findByTenantIdOrderByCreatedAtDesc(Long tenantId);
    List<UserEntity> findByStatusOrderByUpdatedAtDesc(String status);

    // === LIMIT / TOP ===
    Optional<UserEntity> findFirstByTenantIdOrderByCreatedAtDesc(Long tenantId);
    List<UserEntity> findTop10ByStatusOrderByCreatedAtDesc(String status);

    // === COUNT / EXISTS ===
    long countByTenantId(Long tenantId);
    long countByStatus(String status);
    boolean existsByEmail(String email);
    boolean existsByTenantIdAndEmail(Long tenantId, String email);
}
```

**Naming Convention Keywords**:

| Keyword                 | Example                          | SQL Equivalent                     |
|-------------------------|----------------------------------|------------------------------------|
| `And`                   | `findByNameAndStatus`            | `WHERE name = ? AND status = ?`    |
| `Or`                    | `findByNameOrEmail`              | `WHERE name = ? OR email = ?`      |
| `Between`               | `findByCreatedAtBetween`         | `WHERE created_at BETWEEN ? AND ?` |
| `LessThan`              | `findByAgeLessThan`              | `WHERE age < ?`                    |
| `LessThanEqual`         | `findByAgeLessThanEqual`         | `WHERE age <= ?`                   |
| `GreaterThan`           | `findByAgeGreaterThan`           | `WHERE age > ?`                    |
| `GreaterThanEqual`      | `findByAgeGreaterThanEqual`      | `WHERE age >= ?`                   |
| `Like`                  | `findByNameLike`                 | `WHERE name LIKE ?`                |
| `Containing`            | `findByNameContaining`           | `WHERE name LIKE '%?%'`            |
| `StartingWith`          | `findByNameStartingWith`         | `WHERE name LIKE '?%'`             |
| `EndingWith`            | `findByNameEndingWith`           | `WHERE name LIKE '%?'`             |
| `In`                    | `findByStatusIn`                 | `WHERE status IN (?)`              |
| `NotIn`                 | `findByStatusNotIn`              | `WHERE status NOT IN (?)`          |
| `IsNull` / `Null`       | `findByDeletedAtIsNull`          | `WHERE deleted_at IS NULL`         |
| `IsNotNull` / `NotNull` | `findByDeletedAtIsNotNull`       | `WHERE deleted_at IS NOT NULL`     |
| `True` / `False`        | `findByActiveTrue`               | `WHERE active = TRUE`              |
| `OrderBy`               | `findByNameOrderByCreatedAtDesc` | `ORDER BY created_at DESC`         |
| `Not`                   | `findByNameNot`                  | `WHERE name <> ?`                  |
| `First` / `Top`         | `findFirst10ByStatus`            | `LIMIT 10`                         |

### Pattern 2: Default Methods with EasyCriteria (for dynamic queries)

```java
interface _UserRepository extends UserRepository {

    // Dynamic criteria with pagination
    default Page<UserEntity> search(Long tenantId, String keyword, String status, PageRequest page) {
        Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
                .and(EasyCriteria.eq(_Deletable_.DELETED, false));

        if (keyword != null && !keyword.isBlank()) {
            criteria = criteria.and(EasyCriteria.like(Domain.Name, "%" + keyword + "%"));
        }
        if (status != null) {
            criteria = criteria.and(EasyCriteria.eq(Domain.Status, status));
        }

        return findAll(criteria, this.page(page));
    }

    // Exists check (efficient)
    default boolean existsByEmailAndTenant(String email, Long tenantId) {
        return exists(EasyCriteria.eq(Domain.Email, email)
                .and(EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)));
    }

    // Soft-delete aware count
    default long countActiveByTenant(Long tenantId) {
        Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
                .and(EasyCriteria.eq(_Deletable_.DELETED, false))
                .and(EasyCriteria.eq(Domain.Status, "ACTIVE"));
        return count(criteria);
    }
}
```

### Pattern 3: Static SQL Constants with @Query

```java
interface _OrderRepository extends OrderRepository {

    // Inner interface for SQL constants
    interface _SQL {
        String _SUM_AMOUNT_BY_STATUS =
            "SELECT status, SUM(total_amount) FROM USER_ORDER " +
            "WHERE tenant_id = :tenantId AND deleted = false " +
            "GROUP BY status";

        String _UPDATE_STATUS_BATCH =
            "UPDATE USER_ORDER SET status = :newStatus, updated_at = NOW() " +
            "WHERE tenant_id = :tenantId AND status = :oldStatus AND deleted = false";
    }

    @Query(_SQL._SUM_AMOUNT_BY_STATUS)
    List<StatusSummary> sumAmountByStatus(@Param("tenantId") Long tenantId);

    @Modifying
    @Query(_SQL._UPDATE_STATUS_BATCH)
    int updateStatusBatch(@Param("tenantId") Long tenantId,
                          @Param("oldStatus") String oldStatus,
                          @Param("newStatus") String newStatus);
}
```

## 2. EasyCriteria API Reference

**Role**: Type-safe, null-safe `Criteria` builder.

| Method                                   | Description                          |
|------------------------------------------|--------------------------------------|
| `eq(col, val)` / `ne(col, val)`          | Equality / not equal                 |
| `lt(col, val)` / `lte(col, val)`         | Less than / less than or equal       |
| `gt(col, val)` / `gte(col, val)`         | Greater than / greater than or equal |
| `between(col, from, to)`                 | Range                                |
| `like(col, pattern)`                     | LIKE (auto-escapes `%`, `_`)         |
| `startWith(col, prefix)`                 | LIKE 'prefix%'                       |
| `in(col, values)` / `notIn(col, values)` | IN / NOT IN                          |
| `isNull(col)` / `isNotNull(col)`         | Null checks                          |

> **NULL-SAFE**: All methods return `Criteria.empty()` when value is `null`.

## 4. âš ď¸Ź Performance Anti-Patterns

### âťŚ N+1 Problem

```java
// âťŚ WRONG: N+1 queries
List<Order> orders = orderRepo.findByTenantId(tenantId);
for (Order order : orders) {
    User user = userRepo.findById(order.getUserId());  // N queries!
}

// âś… CORRECT: Single query with join or batch fetch
// Option 1: Fetch users in batch
List<Order> orders = orderRepo.findByTenantId(tenantId);
Set<Long> userIds = orders.stream().map(Order::getUserId).collect(Collectors.toSet());
Map<Long, User> userMap = userRepo.findByIdIn(userIds).stream()
        .collect(Collectors.toMap(User::getId, Function.identity()));

// Option 2: Use projection/view in trait extension
@Query("SELECT o.*, u.name as user_name FROM USER_ORDER o JOIN USER u ON o.user_id = u.id WHERE o.tenant_id = :tenantId")
List<OrderWithUser> findWithUserByTenantId(@Param("tenantId") Long tenantId);
```

### âťŚ Inefficient Exists Check

```java
// âťŚ WRONG: Fetches entire entity
boolean exists = userRepo.findByEmail(email).isPresent();

// âś… CORRECT: Uses EXISTS query
boolean exists = userRepo.existsByEmail(email);
// Or with EasyCriteria:
boolean exists = userRepo.exists(EasyCriteria.eq(Domain.Email, email));
```

### âťŚ Unbounded Result Set

```java
// âťŚ WRONG: No limit
List<Order> orders = orderRepo.findByTenantId(tenantId);  // Could be millions!

// âś… CORRECT: Use pagination
Page<Order> orders = orderRepo.findByTenantId(tenantId, PageRequest.of(0, 100));
// Or use Top/First
List<Order> orders = orderRepo.findTop100ByTenantIdOrderByCreatedAtDesc(tenantId);
```

### âťŚ Missing Soft-Delete Filter

```java
// âťŚ WRONG: Includes deleted records
List<User> users = userRepo.findByTenantId(tenantId);

// âś… CORRECT: Filter deleted
List<User> users = userRepo.findByTenantIdAndDeletedFalse(tenantId);
// Or in default method:
Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
        .and(EasyCriteria.eq(_Deletable_.DELETED, false));
```

## 5. DSL Column Access

Columns are static constants in nested `Domain` interface:

```java
// Generated DSL
public interface UserDSL extends DSL<User> {
    interface Domain {
        ColumnMix Id     = _Identifiable_.ID;
        ColumnMix Email  = ColumnMix.of(table, new Column().setName("EMAIL"));
        ColumnMix Status = ColumnMix.of(table, new Column().setName("STATUS"));
    }
    interface _Deletable_ extends Deletable {
        ColumnMix DELETED = ColumnMix.of(table, new Column().setName("DELETED"));
    }
}

// Usage in trait extension
default User findByEmail(String email) {
    return findOne(EasyCriteria.eq(Domain.Email, email)).orElse(null);
}
```

## 6. Complete Example

```java
// === Trait Extension ===
interface _OrderRepository extends OrderRepository {

    // Pattern 1: Spring Data Query Methods
    List<OrderEntity> findByTenantIdAndDeletedFalseOrderByCreatedAtDesc(Long tenantId);
    List<OrderEntity> findByTenantIdAndStatusIn(Long tenantId, List<String> statuses);
    Optional<OrderEntity> findFirstByTenantIdAndUserIdOrderByCreatedAtDesc(Long tenantId, Long userId);
    boolean existsByTenantIdAndOrderNo(Long tenantId, String orderNo);
    long countByTenantIdAndStatus(Long tenantId, String status);

    // Pattern 2: Default Methods with EasyCriteria
    default Page<OrderEntity> search(Long tenantId, String keyword, String status, PageRequest page) {
        Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
                .and(EasyCriteria.eq(_Deletable_.DELETED, false));

        if (keyword != null && !keyword.isBlank()) {
            criteria = criteria.and(EasyCriteria.like(Domain.OrderNo, "%" + keyword + "%"));
        }
        if (status != null) {
            criteria = criteria.and(EasyCriteria.eq(Domain.Status, status));
        }

        return findAll(criteria, this.page(page));
    }

    // Pattern 3: Static SQL
    interface _SQL {
        String _SUM_BY_DATE =
            "SELECT DATE(created_at) as date, SUM(total_amount) as total " +
            "FROM USER_ORDER WHERE tenant_id = :tenantId " +
            "AND created_at BETWEEN :start AND :end AND deleted = false " +
            "GROUP BY DATE(created_at)";
    }

    @Query(_SQL._SUM_BY_DATE)
    List<DailySummary> sumByDateRange(@Param("tenantId") Long tenantId,
                                      @Param("start") LocalDate start,
                                      @Param("end") LocalDate end);
}

// === Service Layer ===
@Service
public class OrderService {
    @Autowired private _OrderRepository orderRepo;

    public Page<OrderEntity> search(Long tenantId, String keyword, String status, PageRequest page) {
        return orderRepo.search(tenantId, keyword, status, page);
    }

    public boolean existsOrderNo(Long tenantId, String orderNo) {
        return orderRepo.existsByTenantIdAndOrderNo(tenantId, orderNo);
    }
}
```

## 7. Best Practices Summary

| Practice | Why |
|----------|-----|
| **DO NOT redefine ListCrudRepository methods** | save/findById/findAll/delete already provided |
| **DO NOT redefine HopeJdbc methods** | findOne/exists/count with Criteria already provided |
| **Use Query Methods for simple queries** | Auto-implemented, no code needed |
| **Use EasyCriteria for dynamic queries** | Type-safe, null-safe |
| **Use `exists()` instead of `findOne().isPresent()`** | Efficient EXISTS query |
| **Always add pagination for list queries** | Prevent unbounded result sets |
| **Filter soft-delete in queries** | `deleted = false` or `findByXxxAndDeletedFalse` |
| **Define SQL constants in `_SQL` interface** | Centralized, maintainable |
| **Run `wire` after proto changes** | Keep DSL/repository in sync |
| **NEVER write query logic in service** | All queries in trait extension |
