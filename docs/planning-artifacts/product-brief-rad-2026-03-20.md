## Executive Summary

RAD 是一个基于 ApiHug 平台的快速应用开发模板，旨在解决企业级应用开发中基础设施功能重复建设的问题。通过预置注册登录、客户管理、租户管理、RBAC 授权、访问记录等标准功能，RAD 将新项目启动时间从 2 周压缩到分钟级，让开发者能够专注于核心业务逻辑而非重复造轮子。

RAD 采用分版本策略：社区版服务于独立开发者和小团队，免费开源；企业版提供高级安全特性、SSO 集成和技术支持。通过深度集成 ApiHug 框架，RAD 填补了框架层与应用开发之间的空白，成为开发者的"时间机器"。

## Core Vision

### Problem Statement

ApiHug 作为框架 + 工具链不提供应用级实现，导致每个新项目都需要重复开发注册登录、客户管理、租户管理、角色授权、访问记录等基础功能。这些工作技术含量不高但必不可少，通常需要至少 2 周时间，成为项目启动的拦路虎。

不同用户群体面临不同痛点：
- **独立开发者**：时间就是金钱，希望快速开始业务开发
- **小团队**：需要标准化起点，确保团队协作效率
- **企业团队**：关注安全合规、数据隔离和技术支持

### Problem Impact

- 开发者陷入重复劳动，无法快速进入核心业务开发
- 项目启动周期长（2 周+），交付压力大
- 基础功能实现质量参差不齐，存在安全隐患
- 缺乏标准化导致技术债务累积

### Why Existing Solutions Fall Short

| 方案 | 不足 |
|------|------|
| ApiHug 原生 | 定位框架层，不涉及应用实现 |
| JHipster | 需要大量适配才能与 ApiHug 集成 |
| RuoYi | 架构和技术栈不匹配，改造成本高 |
| 手动开发 | 耗时 2 周+，质量不稳定 |

### Proposed Solution

创建一个标准的 RAD 模板，采用分版本策略：

**社区版（免费开源）：**
- 注册登录、找回密码
- 客户（用户）管理
- 基础 RBAC（角色 + 权限）
- 单租户模式

**企业版（付费）：**
- 多租户管理（Schema 级隔离）
- 高级 RBAC（数据权限 + 租户权限）
- 审计日志管理（导出 + 合规）
- SSO 集成（SAML/OAuth）
- 技术支持和 SLA

**技术特性：**
- 多租户隔离：Schema 级隔离，安全可靠
- 权限分层：角色→资源→数据三层设计
- 审计优化：异步写入 + 分表策略
- 文档先行：每个功能都有示例和说明

## MVP Scope

### Core Features (Community Edition - Backend Only)

**基础设施（ApiHug 原生）：**
- 多租户管理（Schema 级隔离）
- 审计日志（访问记录自动采集）
- JWT 认证

**核心业务 API：**
- 注册登录 API
- 找回密码 API
- 客户（用户）管理 API（CRUD + 搜索）
- RBAC API（角色 + 权限 + 菜单管理）

**开发体验：**
- 5 分钟启动脚本（gradlew bootRun）
- API 文档（Swagger/OpenAPI 自动生成）
- 快速开始文档（< 10 分钟阅读）
- API 测试示例（Postman Collection）

### Out of Scope for MVP

**V2.0 考虑：**
- 管理后台 UI（Vue 3 + Vben Admin）
- 高级数据权限（行级/列级）
- 工作流引擎
- 报表生成器

**企业版专属：**
- SSO 集成（SAML/OAuth）
- SLA 技术支持
- 定制开发服务

**暂不考虑：**
- 移动端 App
- 低代码可视化编辑器

**用户采用：**
