## Story

作为平台管理员（`CustomerPlatformTypeEnum = OWNER/MANAGER`），
我希望能够管理平台成员（加入、移除、冻结/解冻），
以便平台侧可以独立管理拥有平台访问权限的人员，与租户（Tenant）体系完全隔离。

## 业务背景

### 核心概念说明

本系统存在两套独立的成员体系，必须**严格隔离**：

| 概念 | 实体 | 作用域 | 说明 |
|------|------|--------|------|
| **租户成员** | `TenantMemberEntity` (SYS_TENANT_MEMBER) | 某个具体 Tenant 内 | 客户加入某公司/组织，在该租户下工作 |
| **平台成员** | `PlatformMemberEntity` (SYS_PLATFORM_MEMBER) — **新建** | 全局平台层 | 客户加入平台团队，拥有平台级管理能力 |

**关键区别：**
- 租户成员的作用域是 `tenant_id`，每个租户管理自己的成员
- 平台成员的作用域是全局，没有 `tenant_id` 概念
- 一个 Customer 可以同时是某租户的成员，也是平台成员
- 平台成员身份通过 `CustomerEntity.platform_type` 字段标记（非 `NA` 即为平台成员）

### `CustomerPlatformTypeEnum` 现状

```proto
// 文件: com/apihug/rad/infra/customer/constant.proto
enum CustomerPlatformTypeEnum {
  MEMBER = 0;   // code:1 — 平台员工（普通平台成员）
  MANAGER = 1;  // code:1 — 平台管理员 ⚠️ BUG: code 与 MEMBER 重复！
  OWNER = 2;    // code:2 — 平台拥有者
}
```

**注意：** 还有一个 `NA` 值尚未定义在枚举中，但 `domain.proto` 中 `default_value: "NA"` 已使用。
需要在 `CustomerPlatformTypeEnum` 中补充 `NA` 作为默认值，同时修复 `MANAGER` 的 `code` 重复 BUG。

### 状态语义说明

- `CustomerEntity.platform_type = NA`：普通客户，不属于平台成员
- `CustomerEntity.platform_type = MEMBER/MANAGER/OWNER`：该客户是平台成员
- 加入平台：创建 `PlatformMemberEntity` 记录 + 更新 `CustomerEntity.platform_type`
- 移除平台：删除/停用 `PlatformMemberEntity` + 将 `CustomerEntity.platform_type` 重置为 `NA`
- 冻结：`PlatformMemberEntity.status = PM_LOCKED`，不直接改变 `platform_type`

## Acceptance Criteria

**AC1: 查看平台成员列表**
- Given 已登录的平台管理员（`platform_type IN [MANAGER, OWNER]`）
- When 请求获取平台成员列表（支持分页、状态筛选、关键词搜索）
- Then 返回 200 OK，包含平台成员摘要列表（分页）
- And 每条记录含：客户基本信息 + 平台角色 + 平台成员状态 + 加入时间

**AC2: 添加平台成员**
- Given 平台 OWNER 或 MANAGER 发送添加成员请求
- When 目标 Customer 存在且尚未加入平台（`platform_type = NA`）
- Then 创建 `PlatformMemberEntity` 记录
- And 同步更新 `CustomerEntity.platform_type` 为请求中指定的角色（MEMBER/MANAGER）
- And 返回 201 Created 或 200 OK

**AC3: 移除平台成员**
- Given 平台 OWNER 或 MANAGER 发送移除请求
- When 目标平台成员存在且状态为非 OWNER（OWNER 不允许被移除）
- Then 将 `PlatformMemberEntity.status` 更新为 `PM_INACTIVE`
- And 将 `CustomerEntity.platform_type` 重置为 `NA`
- And 返回 200/204

**AC4: 冻结/解冻平台成员**
- Given 平台 OWNER 或 MANAGER 发送冻结/解冻请求
- When 目标平台成员存在
- Then 切换 `PlatformMemberEntity.status`：`PM_ACTIVE ↔ PM_LOCKED`
- And `CustomerEntity.platform_type` 不变（冻结只影响访问能力，身份保留）
- And 返回 200/204

**AC5: 数据一致性约束**
- `CustomerEntity.platform_type` 与 `PlatformMemberEntity` 记录必须保持同步
- 同一 Customer 在 `SYS_PLATFORM_MEMBER` 中只能有一条有效记录（唯一约束：customer_id）
- OWNER 角色的平台成员不允许被移除（保护机制）

**AC6: 更新平台成员角色**
- Given 平台 OWNER 或 MANAGER 发送更新角色请求
- When 目标平台成员存在且状态为 PM_ACTIVE 或 PM_LOCKED
- Then 更新 `PlatformMemberEntity.platform_role` 为请求中指定的角色
- And 同步更新 `CustomerEntity.platform_type` 为新角色
- And PM_INACTIVE 状态的成员不允许更新（返回 400）
- And 返回 200 OK

### CustomerPlatformTypeEnum BUG 修复说明

**当前 Bug（必须修复后才能实现功能）：**
```proto
// 当前问题：MANAGER 和 MEMBER 的 code 值均为 1（重复！）
```

**修复方案：**
```proto
enum CustomerPlatformTypeEnum {

  MANAGER = 2 [(hope.constant.field) = { code: 2, message: "platform manager", message2: "平台管理员" }];
```

### 边界条件处理

1. **重复加入保护**：Customer 已是平台成员（`platform_type != NA` 或 SYS_PLATFORM_MEMBER 存在有效记录），返回 `PLATFORM_MEMBER_ALREADY_EXISTS` 错误
2. **OWNER 保护**：`platform_role = OWNER` 的成员不允许被移除，防止系统无管理员
3. **操作权限**：写操作（添加/移除/冻结）需要 `CustomerPlatformTypeEnum IN [OWNER, MANAGER]`；读操作至少需要 `MEMBER`
4. **数据一致性**：添加/移除操作涉及两张表（`SYS_PLATFORM_MEMBER` + `SYS_CUSTOMER`），需在同一事务中处理

### 与 TenantMember 登录流程的关系

- JWT claims 中将携带 `platform_type` 字段（标识平台身份）

- **不新增独立的 PlatformMember 登录流程**，复用现有 Customer 登录

## ApiHug Developer Guardrails

### Proto Design Rules (if applicable)

- **Pageable API**：`GetPlatformMembers` Request 无需 page/size/sort 字段，使用 `pageable: true`
- **Request Message 设计**：
  - `AddPlatformMember`：有 customer_id + platform_role 2个字段 → 使用 `google.protobuf.Empty` + `parameters{}`
  - `RemovePlatformMember`：只有 memberId path param → 使用 `google.protobuf.Empty` + path parameters
  - `TogglePlatformMemberFreeze`：只有 memberId path param → 使用 `google.protobuf.Empty` + path parameters
- **SQL Reserved Words**：`status` 必须用列名 `STATUS_CODE`（参考 TenantMemberEntity 中的 `STATUS_CODE`）
- **Table Prefix**：`SYS_PLATFORM_MEMBER` ✓（SYS_ 前缀）
- **Enum Persistence**：使用 `enum_type: STRING`（禁止 ORDINAL）
- **Error Codes**：使用 `http_status` 枚举名（如 `NOT_FOUND`），不用数字

### Implementation Rules (if applicable)

- **ServiceImpl**：Wire 生成骨架后填充方法体；NEVER 手写 ServiceImpl 类头/注解/构造器
- **Repository**：所有查询逻辑（按 customer_id 查找、分页查询）MUST 在 trait repository 中，不在 ServiceImpl 中写查询
- **Error Handling**：
  - DomainService 抛出 `HopeErrorDetailException`（使用 `PlatformMemberErrorEnum`）
  - ServiceImpl 使用 `builder.error(PlatformMemberErrorEnum.PLATFORM_MEMBER_NOT_FOUND)` 模式
- **双表操作事务**：添加/移除平台成员时涉及 `SYS_PLATFORM_MEMBER` 和 `SYS_CUSTOMER` 两张表，须在同一事务内完成
- **OWNER 保护**：在 DomainService 中检查，不是在 proto 中

### Frontend Rules (if applicable)

本 story 暂不包含前端实现，后续可按以下模式添加：
- LIST_PAGE: `views/platform/member/index.vue`
- FORM_PAGE: `views/platform/member/components/AddMemberModal.vue`
- Auto-Router: 运行 `gradlew :ui_module:ui` 自动生成
